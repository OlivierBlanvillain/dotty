6 /**
7  * Core BTypes and some other definitions. The initialization of these definitions requies access
8  * to symbols / types (global).
9  *
10  * The symbols used to initialize the ClassBTypes may change from one compiler run to the next. To
11  * make sure the definitions are consistent with the symbols in the current run, the
12  * `intializeCoreBTypes` method in BTypesFromSymbols creates a new instance of CoreBTypes in each
13  * compiler run.
14  *
15  * The class BTypesFromSymbols does not directly reference CoreBTypes, but CoreBTypesProxy. The
16  * reason is that having a `var bTypes: CoreBTypes` would not allow `import bTypes._`. Instead, the
17  * proxy class holds a `CoreBTypes` in a variable field and forwards to this instance.
18  *
19  * The definitions in `CoreBTypes` need to be lazy vals to break an initialization cycle. When
20  * creating a new instance to assign to the proxy, the `classBTypeFromSymbol` invoked in the
21  * constructor will actucally go through the proxy. The lazy vals make sure the instance is assigned
22  * in the proxy before the fields are initialized.
23  *
24  * Note: if we did not re-create the core BTypes on each compiler run, BType.classBTypeFromInternalNameMap
25  * could not be a perRunCache anymore: the classes defeined here need to be in that map, they are
26  * added when the ClassBTypes are created. The per run cache removes them, so they would be missing
27  * in the second run.
28  */

81   /**
82    * Maps the method symbol for a box method to the boxed type of the result. For example, the
83    * method symbol for `Byte.box()` is mapped to the ClassBType `java/lang/Byte`.
84    */
85   lazy val boxResultType: Map[Symbol, ClassBType] = {
86     for ((valueClassSym, boxMethodSym) <- boxMethods)
87     yield boxMethodSym -> boxedClassOfPrimitive(primitiveTypeMap(valueClassSym))
88   }

94     for ((valueClassSym, unboxMethodSym) <- unboxMethods)
