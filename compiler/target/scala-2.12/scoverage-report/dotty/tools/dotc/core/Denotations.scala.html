27 /** Denotations represent the meaning of symbols and named types.
28  *  The following diagram shows how the principal types of denotations
29  *  and their denoting entities relate to each other. Lines ending in
30  *  a down-arrow `v` are member methods. The two methods shown in the diagram are
31  *  "symbol" and "deref". Both methods are parameterized by the current context,
32  *  and are effectively indexed by current period.
33  *
34  *  Lines ending in a horizontal line mean subtying (right is a subtype of left).
35  *
36  *  NamedType
37  *    |                                          Symbol---------ClassSymbol
38  *    |                                            |                |
39  *    | denot                                      | denot          | denot
40  *    v                                            v                v
41  *  Denotation-+-----SingleDenotation-+------SymDenotation-+----ClassDenotation
42  *             |                      |
43  *             +-----MultiDenotation  |
44  *                                    |
45  *                                    +--UniqueRefDenotation
46  *                                    +--JointRefDenotation

48  *  Here's a short summary of the classes in this diagram.
49  *
50  *  NamedType                A type consisting of a prefix type and a name, with fields
51  *                              prefix: Type
52  *                              name: Name
53  *                           It has two subtypes: TermRef and TypeRef
54  *  Symbol                   A label for a definition or declaration in one compiler run
55  *  ClassSymbol              A symbol representing a class
56  *  Denotation               The meaning of a named type or symbol during a period
57  *  MultiDenotation          A denotation representing several overloaded members

353      *
354      *  NoDenotations are dropped. MultiDenotations are handled by merging
355      *  parts with same signatures. SingleDenotations with equal signatures
356      *  are joined as follows:
357      *
358      *  In a first step, consider only those denotations which have symbols
359      *  that are accessible from prefix `pre`.
360      *
361      *  If there are several such denotations, try to pick one by applying the following
362      *  three precedence rules in decreasing order of priority:
363      *
364      *  1. Prefer denotations with more specific infos.
365      *  2. If infos are equally specific, prefer denotations with concrete symbols over denotations
366      *     with abstract symbols.

368      *     prefer denotations with symbols defined in subclasses
369      *     over denotations with symbols defined in proper superclasses.
370      *
371      *  If there is exactly one (preferred) accessible denotation, return it.

374      *  with one of the operand symbols (unspecified which one), and an info which
375      *  is the intersection (using `&` or `safe_&` if `safeIntersection` is true)
376      *  of the infos of the operand denotations.
377      *
378      *  If SingleDenotations with different signatures are joined, return NoDenotation.

383        *  the possibility of raising a merge error.

385       def infoMeet(tp1: Type, tp2: Type): Type = {
386         if (tp1 eq tp2) tp1
387         else tp1 match {
388           case tp1: TypeBounds =>
389             tp2 match {
390               case tp2: TypeBounds => if (safeIntersection) tp1 safe_& tp2 else tp1 & tp2
391               case tp2: ClassInfo if tp1 contains tp2 => tp2
392               case _ => mergeConflict(tp1, tp2)
393             }
394           case tp1: ClassInfo =>
395             tp2 match {
396               case tp2: ClassInfo if tp1.cls eq tp2.cls => tp1.derivedClassInfo(tp1.prefix & tp2.prefix)
397               case tp2: TypeBounds if tp2 contains tp1 => tp1
398               case _ => mergeConflict(tp1, tp2)
399             }
400           case tp1: MethodOrPoly =>
401             tp2 match {
402               case tp2: MethodOrPoly =>
403                 // Two remedial strategies:
404                 //
405                 //  1. Prefer method types over poly types. This is necessary to handle
406                 //     overloaded definitions like the following
407                 //
408                 //        def ++ [B >: A](xs: C[B]): D[B]
409                 //        def ++ (xs: C[A]): D[A]
410                 //
411                 //     (Code like this is found in the collection strawman)
412                 //
413                 // 2. In the case of two method types or two polytypes with matching
414                 //    parameters and implicit status, merge corresppnding parameter
415                 //    and result types.
416                 if (tp1.isInstanceOf[PolyType] && tp2.isInstanceOf[MethodType]) tp2
417                 else if (tp2.isInstanceOf[PolyType] && tp1.isInstanceOf[MethodType]) tp1
418                 else if (ctx.typeComparer.matchingParams(tp1, tp2) &&

479          *   2. sym1 is concrete and sym2 is abstract

495               (!sym2.isAsConcrete(sym1) ||

498                 sym1.is(Method) && !sym2.is(Method)) ||
499               sym1.info.isErroneous)

503           preferSym(sym1, sym2) &&

508           else if (preferSym(sym2, sym1)) denot2

510

512         else {

514           if (sym1Accessible && prefer(sym1, sym2, info1, info2)) denot1
515           else if (sym1Accessible && sym2.exists && !sym2Accessible) denot1
516           else if (sym2Accessible && sym1.exists && !sym1Accessible) denot2
517           else if (isDoubleDef(sym1, sym2)) handleDoubleDef
518           else {
519             val sym =
520               if (preferSym(sym2, sym1)) sym2
521               else sym1
522             val jointInfo =
523               try infoMeet(info1, info2)
524               catch {

526                   if (pre.widen.classSymbol.is(Scala2x) || ctx.scala2Mode)

528                   // compare with way merge is performed in SymDenotation#computeMembersNamed
529                   else

532             new JointRefDenotation(sym, jointInfo, denot1.validFor & denot2.validFor)
533           }
534         }
535       }

546       }

561             case tp2: TypeBounds => tp1 | tp2
562             case tp2: ClassInfo if tp1 contains tp2 => tp1
563             case _ => mergeConflict(tp1, tp2)
564           }
565         case tp1: ClassInfo =>
566           tp2 match {
567             case tp2: ClassInfo if tp1.cls eq tp2.cls => tp1.derivedClassInfo(tp1.prefix | tp2.prefix)
568             case tp2: TypeBounds if tp2 contains tp1 => tp2
569             case _ => mergeConflict(tp1, tp2)
570           }
571         case tp1: MethodOrPoly =>
572           tp2 match {
573             case tp2: MethodOrPoly
574             if ctx.typeComparer.matchingParams(tp1, tp2) &&
575                tp1.isImplicitMethod == tp2.isImplicitMethod =>
576               tp1.derivedLambdaType(
577                 mergeParamNames(tp1, tp2), tp1.paramInfos,
578                 tp1.resultType | tp2.resultType.subst(tp2, tp1))
579             case _ =>
580               mergeConflict(tp1, tp2)
581           }
582         case _ =>
583           tp1 | tp2
584       }
585 
586       def unionDenot(denot1: SingleDenotation, denot2: SingleDenotation): Denotation =
587         if (denot1.matches(denot2)) {
588           val sym1 = denot1.symbol
589           val sym2 = denot2.symbol
590           val info1 = denot1.info
591           val info2 = denot2.info
592           val sameSym = sym1 eq sym2
593           if (sameSym && (info1 frozen_<:< info2)) denot2
594           else if (sameSym && (info2 frozen_<:< info1)) denot1
595           else {
596             val jointSym =
597               if (sameSym) sym1
598               else {
599                 val owner2 = if (sym2 ne NoSymbol) sym2.owner else NoSymbol
600                 /** Determine a symbol which is overridden by both sym1 and sym2.
601                  *  Preference is given to accessible symbols.
602                  */
603                 def lubSym(overrides: Iterator[Symbol], previous: Symbol): Symbol =
604                   if (!overrides.hasNext) previous
605                   else {
606                     val candidate = overrides.next()
607                     if (owner2 derivesFrom candidate.owner)
608                       if (candidate isAccessibleFrom pre) candidate
609                       else lubSym(overrides, previous orElse candidate)
610                     else
611                       lubSym(overrides, previous)
612                   }
613                 lubSym(sym1.allOverriddenSymbols, NoSymbol)
614               }
615             new JointRefDenotation(
616                 jointSym, infoJoin(info1, info2), denot1.validFor & denot2.validFor)
617           }
618         }
619         else NoDenotation
620 
621       if (this eq that) this
622       else if (!this.exists) this
623       else if (!that.exists) that
624       else this match {
625         case denot1 @ MultiDenotation(denot11, denot12) =>
626           denot1.derivedUnionDenotation(denot11 | (that, pre), denot12 | (that, pre))
627         case denot1: SingleDenotation =>
628           that match {
629             case denot2 @ MultiDenotation(denot21, denot22) =>
630               denot2.derivedUnionDenotation(this | (denot21, pre), this | (denot22, pre))
631             case denot2: SingleDenotation =>
632               unionDenot(denot1, denot2)
633           }
634       }
635     }

824     /** Produce a denotation that is valid for the given context.
825      *  Usually called when !(validFor contains ctx.period)
826      *  (even though this is not a precondition).
827      *  If the runId of the context is the same as runId of this denotation,
828      *  the right flock member is located, or, if it does not exist yet,
829      *  created by invoking a transformer (@See Transformers).
830      *  If the runId's differ, but this denotation is a SymDenotation
831      *  and its toplevel owner class or module
832      *  is still a member of its enclosing package, then the whole flock
833      *  is brought forward to be valid in the new runId. Otherwise
834      *  the symbol is stale, which constitutes an internal error.
835      */
836     def current(implicit ctx: Context): SingleDenotation = {
837       val currentPeriod = ctx.period
838       val valid = myValidFor
839       if (valid.code <= 0) {
840         // can happen if we sit on a stale denotation which has been replaced
841         // wholesale by an installAfter; in this case, proceed to the next
842         // denotation and try again.
843         val nxt = nextDefined
844         if (nxt.validFor != Nowhere) return nxt
845         assert(false, this)
846       }
847 
848       if (valid.runId != currentPeriod.runId)
849         if (exists) initial.bringForward().current
850         else this
851       else {
852         var cur = this
853         if (currentPeriod.code > valid.code) {
854           // search for containing period as long as nextInRun increases.
855           var next = nextInRun
856           while (next.validFor.code > valid.code && !(next.validFor contains currentPeriod)) {
857             cur = next
858             next = next.nextInRun
859           }
860           if (next.validFor.code > valid.code) {
861             // in this case, next.validFor contains currentPeriod
862             cur = next
863             cur
864           } else {
865             //println(s"might need new denot for $cur, valid for ${cur.validFor} at $currentPeriod")
866             // not found, cur points to highest existing variant
867             val nextTransformerId = ctx.nextDenotTransformerId(cur.validFor.lastPhaseId)
868             if (currentPeriod.lastPhaseId <= nextTransformerId)
869               cur.validFor = Period(currentPeriod.runId, cur.validFor.firstPhaseId, nextTransformerId)
870             else {
871               var startPid = nextTransformerId + 1
872               val transformer = ctx.denotTransformers(nextTransformerId)
873               //println(s"transforming $this with $transformer")
874               try {
875                 next = transformer.transform(cur)(ctx.withPhase(transformer))
876               } catch {
877                 case ex: CyclicReference =>
878                   println(s"error while transforming $this") // DEBUG
879                   throw ex
880               }
881               if (next eq cur)
882                 startPid = cur.validFor.firstPhaseId
883               else {
884                 next match {
885                   case next: ClassDenotation =>
886                     assert(!next.is(Package), s"illegal transformation of package denotation by transformer ${ctx.withPhase(transformer).phase}")
887                   case _ =>
888                 }
889                 next.insertAfter(cur)
890                 cur = next
891               }
892               cur.validFor = Period(currentPeriod.runId, startPid, transformer.lastPhaseId)
893               //printPeriods(cur)
894               //println(s"new denot: $cur, valid for ${cur.validFor}")
895             }
896             cur.current // multiple transformations could be required
897           }
898         } else {
899           // currentPeriod < end of valid; in this case a version must exist
900           // but to be defensive we check for infinite loop anyway
901           var cnt = 0
902           while (!(cur.validFor contains currentPeriod)) {
903             //println(s"searching: $cur at $currentPeriod, valid for ${cur.validFor}")
904             cur = cur.nextInRun
905             // Note: One might be tempted to add a `prev` field to get to the new denotation
906             // more directly here. I tried that, but it degrades rather than improves
907             // performance: Test setup: Compile everything in dotc and immediate subdirectories
908             // 10 times. Best out of 10: 18154ms with `prev` field, 17777ms without.
909             cnt += 1
910             if (cnt > MaxPossiblePhaseId)
911               return current(ctx.withPhase(coveredInterval.firstPhaseId))
912           }
913           cur
914         }
915       }
916     }
